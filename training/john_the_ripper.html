OFFLINE CREDENTIAL ATTACKS (JOHN THE RIPPER)

Why stolen hashes are more dangerous than live logins

1. What Is John the Ripper?

John the Ripper is an offline password cracking tool. It attempts to recover plaintext passwords from stolen password hashes after a breach.

  • No rate limiting
  • No detection by the victim
  • Speed depends only on hardware

2. Why Offline Attacks Are So Dangerous

  • Attackers can try billions of guesses per second
  • No account lockouts
  • No alerts to defenders

Once hashes are leaked, the system is already compromised.

3. Common Hash Types John Cracks

MD5

  • Hash size: 128-bit
  • Speed: Extremely fast
  • Crack time: Seconds to minutes
  • Why dangerous: Designed for speed, not security

SHA-1

  • Hash size: 160-bit
  • Speed: Very fast
  • Crack time: Minutes to hours
  • Known breach: LinkedIn (2012)

SHA-256 / SHA-512

  • Speed: Fast
  • Crack time: Hours to days
  • Issue: Still too fast for passwords

NTLM

  • Used by: Windows environments
  • Speed: Extremely fast
  • Risk: Enables lateral movement

bcrypt

  • Speed: Very slow
  • Crack time: Months to years
  • Defense: Built for password storage
